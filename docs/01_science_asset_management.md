# 科学资产管理方法

科学资产管理的核心不是积累链接，而是把外部资料转化为个人工作流和 MatOS 都能复用、验证、交付的能力单元。

## 基本判断

- 网页不是学习材料，而是能力证据。
- example 不是教程，而是 benchmark。
- 实习生不是照着跑代码，而是把 benchmark 封装为 scientific capability。
- 负责人不是亲自学习每个软件，而是定义能力、拆解任务、验收科学正确性。

## 从资料到能力的流程

1. 识别资料来源：记录官方网页、软件 example、论文、案例库或工程标准。
2. 抽象问题类型：明确该资料解决的是燃烧、CFD、多物理场、网格、结构力学还是数值求解问题。
3. 提取物理模型：识别控制方程、反应机理、材料模型、边界条件、网格要求和求解器设定。
4. 固化输入输出：把隐含参数转化为 schema，把结果转化为结构化输出和报告。
5. 形成 benchmark：确定默认案例、参考输出、可重复运行方式和误差容许范围。
6. 定义自动验收：检查运行成功、收敛性、文件产物、物理趋势和结果范围。
7. 设计集成路径：给出 package entrypoint、workflow stage、输入契约、输出契约和后处理方式；必要时说明个人 workflow 与 MatOS 的差异。

## 任务拆解原则

每个任务必须同时满足三类读者：

- 实习生可执行：任务边界清楚，输入输出明确，知道要交付什么。
- 负责人可验收：有 benchmark、自动检查、趋势解释和报告模板。
- 个人或 MatOS 可集成：有 schema、entrypoint、artifact 结构和 workflow 接入方式。

## 验收关注点

科学计算能力不能只证明“程序能跑”。验收必须覆盖：

- 求解器选择是否匹配问题。
- 物理模型和边界条件是否明确。
- 网格或离散设置是否足以支撑结果。
- 收敛过程是否可追踪。
- 关键结果是否满足物理正确性。
- 参数扰动趋势是否合理。
- 输出是否能被个人 workflow 或 MatOS 自动读取、比较和汇报。
