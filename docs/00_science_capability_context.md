# 科学能力资产背景

本仓库服务于个人科研自动化与 MatOS 集成两类目标。MatOS 是一个面向科学计算、材料、能源、化工、CAE 和多物理场仿真的科研操作系统；个人工作流也需要同一类可复现、可验证、可扩展的能力资产。这里的核心不是简单调用软件，而是组织 Knowledge、Tools、Skills、Agents、Workflows，将科学问题转化为可复用的自动化科研流程。

## 为什么需要科学能力资产注册层

科学计算软件的价值不在于“会打开软件”或“能运行一个例子”，而在于能把特定问题稳定地转化为输入、模型、求解、后处理、验证和复用流程。个人工作流和 MatOS 都需要把这些能力沉淀为结构化资产：

- Knowledge：问题背景、物理模型、软件适用边界和参考资料。
- Tools：Cantera、OpenFOAM、Fluent、COMSOL、Abaqus、Gmsh、ParaView 等可调用软件工具。
- Skills：围绕燃烧、CFD、多物理场、网格、数值求解等领域的可复用工作方法。
- Agents：能够读取资产卡、拆解任务、生成代码、运行验收和汇报风险的自动化执行者。
- Workflows：从问题定义到 benchmark、capability、自动验收和报告的闭环流程。

## 资产化目标

本仓库服务于中立的科学能力建设。每个外部网页、软件 example 或 benchmark 都应被处理成可追踪的能力资产；它可以被个人 agent、本地项目或 MatOS 消费，但不默认代表资产所有权归属于 MatOS。
